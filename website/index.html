ABOUT EAGLE

Apache Eagle is an Open Source Monitoring solution, contributed by eBay Inc, to instantly identify access to sensitive data, recognize attacks, malicious activities in Hadoop and take actions in real time.

Watch the Intro Video

Eagle has been accepted as an Apache Incubator Project on Oct 26, 2015.

Eagle secures Hadoop in realtime in 3 steps:

Step 1

Listens to the various user generated signals by streaming through the Hadoop Audit and System logs.

Step 2

Analyzes these signals to detect anomalies based on predefined security policies and machine learning models according to the user's past behavior.

Step 3

Generates Alerts on detection of anomalies and remediates user activity/access if necessary.

Additional Bonus

Eagle also provides an extensible architecture and framework for developers to easily onboard new use cases into the framework.

Ease Of Use

Start using Eagle in few minutes

Eagle can be installed as an Ambari plugin. And you can use one instance of eagle to monitor multiple clusters.

Policy Management

Create complex monitoring policies

Eagle offers wide variety of policies on HDFS, Hive data sets. It is easy to create and deploy policies with few clicks.

Scalability

Scale for Big Data

Eagle processes access events in a scalable way to support billions of events and thousands of policies.

Extensibility

Eagle is extensible at all levels

Easy to integrate with data classification and hadoop access control products. Eagle backend can support data sources other than hadoop.

MODULES

Eagle is built using big data open source products to scale for big data use cases.

Processing Engine

Eagle stream processing API is an abstraction of Apache Storm, extensible with other streaming engine. Eagle processing is built to scale to large number of events and policies.

Alerting Framework

Eagle’s extensible policy engine framework allows developer to plugin a new policy engine with a few lines of codes. Eagle supports WSO2 Siddhi CEP engine.

Machine Learning module

Eagle provides capabilities to define user profiles for Hadoop users based on the user behavior. Eagle uses Spark for data training and Storm for near real-time anomaly detection.

Data Classification

Eagle allows you to easily classify HDFS and Hive data based on sensitivity of the data. Eagle can also be easily extended to import data classification from other tools.

Policy Manager

Eagle policy manager provides a easy to use interface and Restful API for user to define policy with a few clicks. Eagle supports window functions to create complex policies.

Query Service

Eagle provides SQL-like service API to support comprehensive computation for huge set of data, for example filtering, aggregation, histogram, sorting, top, arithmetical expression, pagination etc.

USE CASES

Data loss prevention

Prevent a user with malicious intent to move, copy, delete sensitive data from hadoop.

Active Access monitoring

Monitor every single user access to hadoop data sets in HDFS and Hive in realtime.

User Profiles

Create user profiles based on the user behavior using machine learning algorithms. And use these models to detect and alert on anomalies.

Unauthorized Access

Identify, alert and stop an unauthorized user trying to access sensitive data stored in the hadoop cluster.
